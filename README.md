# fsv - File Status Viewer

一个轻量级的终端文件状态查看器，使用 Rust 开发。实时显示 Git 仓库中所有文件的状态，支持目录折叠、文件编辑等功能。

## 功能特性

- 📁 **完整目录显示** - 显示完整的目录结构，不仅仅是有变更的文件
- 🎨 **Git 状态标识** - 以不同颜色显示文件的 Git 状态
- 🔄 **实时刷新** - 文件变化时自动更新状态显示
- ⌨️ **键盘导航** - 使用方向键进行导航
- 🔍 **快速搜索** - 按 `Ctrl+P` 快速搜索文件
- 🖱️ **鼠标支持** - 支持鼠标点击选择、滚动操作
- 📝 **内置编辑器** - 直接在侧边栏编辑文件内容
- 🔄 **撤销功能** - 支持 Ctrl+Z 撤销操作
- 📊 **Diff 高亮** - 自动高亮显示修改的行
- 🚀 **高性能** - Rust 编译为单二进制文件，低资源占用
- 🖥️ **内置终端** - 底部终端支持执行命令与多 Tab

## Git 状态说明

| 状态 | 符号 | 颜色 | 说明 |
|------|------|------|------|
| Modified | M | 黄色 | 文件已修改 |
| Added | A | 绿色 | 新增文件（已暂存） |
| Deleted | D | 红色 | 删除文件 |
| Untracked | ?? | 灰色 | 未跟踪文件（新建未添加） |
| Renamed | R | 紫色 | 重命名文件 |
| Clean | (空格) | 白色 | 无变更 |

## 安装

### 前置要求

- Rust 1.70+ (使用 `rustup` 安装)

### 构建

```bash
# 克隆或进入项目目录
cd fsv

# 开发构建
cargo build

# 发布构建（优化）
cargo build --release

# 运行
cargo run

# 或运行编译后的二进制文件
./target/release/fsv.exe  # Windows
./target/release/fsv      # Linux/macOS
```

## 使用方法

```bash
# 在当前目录运行
fsv

# 指定工作目录
fsv /path/to/your/project
```

## 显示模式

1. **全部模式** - 显示所有文件（包括未跟踪）
2. **变更模式** - 只显示有变更的文件
3. **已跟踪模式** - 只显示 Git 已跟踪的文件

按 `m` 键在三种模式间切换。

## 搜索功能

按 `Ctrl+P` 打开搜索框，输入文件名或路径关键词快速定位文件。搜索时会自动跳转并选中匹配的文件。

- 按 `Enter` 确认搜索
- 按 `Esc` 取消搜索

## Diff 高亮

在编辑器中修改文件时，会自动高亮显示被修改的行：

- **绿色背景** - 表示该行已被修改
- **绿色行号** - 表示该行是修改的

保存文件后，Git 状态会更新，文件在目录树中会显示为 Modified（黄色）。

## Enter 键行为

Enter 键根据选中的内容智能操作：
- **选中文件时** - 打开编辑器
- **选中目录时** - 展开/收起目录

## 文件状态说明

- **Clean (空格)** - 文件与 Git 仓库中的一致，无变更
- **Modified (M)** - 文件内容已修改
- **Added (A)** - 新文件已暂存（git add）
- **Untracked (??)** - 新建文件，尚未被 Git 跟踪
- **Deleted (D)** - 文件已被删除

## 故障排除

### 问题：中文显示乱码

**解决**：确保终端使用 UTF-8 编码，并使用支持中文的字体。

### 问题：文件状态不更新

**解决**：按 `r` 键手动刷新，或检查文件变化监听是否正常工作。

### 问题：Git 状态不准确

**解决**：确保在 Git 仓库根目录或子目录中运行。可以使用 `git status` 命令验证状态。
